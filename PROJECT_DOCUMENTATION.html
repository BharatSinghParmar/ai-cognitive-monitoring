1. Introduction & Background

1.1 Background & Problem Statement

In the contemporary educational and corporate landscape, remote assessments have become the standard methodology for knowledge verification globally. While Learning Management Systems (LMS) have evolved significantly to distribute and collect examinations, maintaining academic integrity in an unmonitored environment remains a profound challenge.

The core problem extends beyond simple impersonation or "tab-switching." Sophisticated circumvention often bypasses passive locks. A secondary, less addressed problem in modern pedagogy is the psychological impact of assessments. Standard exam systems fail to assess how a student answers a question—measuring only the output, not the cognitive journey, fatigue, or stress incurred. Thus, an ideal system must not only catch cheating but also understand the student's cognitive state.

  • To automate the assessment of unstructured descriptive answers using Lexical & Semantic NLP Similarity Algorithms.

2. Theoretical Framework & Core Concepts

The foundation of this project is deeply rooted in Applied Mathematics, Computer Vision, Cognitive Psychology, and Natural Language Processing. This section explicitly defines the mathematical and conceptual models implemented in the source code.

2.1 Biometric Authentication & Computer Vision

Facial recognition is the cornerstone of the platform’s identity verification. The system utilizes Convolutional Neural Networks (CNN) provided by face-api.js (which wraps TensorFlow.js) to execute mobile-net based face detection.

Theoretical Process:

  1. Detection: The TinyFaceDetector locates the bounding box of the face in the continuous video stream.
  2. Landmark Mapping: The FaceLandmark68Net maps exactly 68 defining topographical coordinates (eyes, nose, jawline, mouth).
  3. Feature Extraction: The FaceRecognitionNet projects these landmarks into a high-dimensional space, outputting a 128-dimensional floating-point array (Face Descriptor).

To verify a student, we calculate the Euclidean Distance between the registered descriptor array (\(p\)) and the live video capture array (\(q\)) in 128-dimensional space:

\( d(p, q) = \sqrt{\sum_{i=1}^{n=128} (q_i - p_i)^2} \)
Formula 2.1: Euclidean Distance for Face Verification

In our system, a threshold distance of 0.6 is mathematically defined. If \( d(p, q) \le 0.6 \), the faces match (same person). If \( d(p, q) > 0.6 \), authentication is denied. This logic is directly translated in backend/routes/userRoutes.js.

2.2 Cognitive Load Theory & Stress Telemetry

Cognitive Load Theory (Sweller, 1988) suggests that working memory has a limited capacity. When cognitive load exceeds capacity (high stress, difficult tasks, or external anxiety like cheating), it manifests physiologically.

A. Visual Stress (Eye Aspect Ratio - EAR)

Using the 68 facial landmarks, we isolate the points around the eyes. The Eye Aspect Ratio (Soukupová & Čech, 2016) is a scalar value mapping the openness of the eye. Six coordinates represent the eye: \( p_1, ..., p_6 \).

\( EAR = \frac{\|p_2 - p_6\| + \|p_3 - p_5\|}{2 \|p_1 - p_4\|} \)
Formula 2.2: Eye Aspect Ratio (EAR)

When the eye closes (blink or drowsiness), the numerator approaches zero. Rapid fluctuations indicate panic, while sustained low EAR indicates drowsiness or distraction. In BehaviorMonitor.js, an EAR below 0.20 adds a massive +40 penalty strictly to the current visual stress metric.

B. Acoustic Stress (Frequency & Amplitude Energy)

The human voice exhibits micro-tremors under stress. Using the Web Audio API AnalyserNode, the system computes the Fast Fourier Transform (FFT) of the audio stream. The root mean square (RMS) amplitude or frequency band averages are tracked. High variance or sudden acoustic volume peaks generate acoustic stress spikes in the Cognitive HUD.

C. Kinetic Dynamics (Keystroke Variance)

Cognitive hesitation translates directly to keyboard dwell time and flight time. By recording the timestamp difference between keydown events, we calculate the localized variance of typing speed over a rolling 10-key buffer. High variance equals cognitive dissonance (unsure answers, or frantically typing copied text).

\( \sigma^2 = \frac{\sum_{i=1}^{n} (x_i - \mu)^2}{n} \)
Formula 2.3: Variance computation for typing intervals

2.3 Natural Language Processing (NLP) in Grading

To automatically grade descriptive (essay) questions, the system must measure how closely a student's unstructured text aligns semantically and lexically with an admin-defined "Reference Answer."

The similarity.js utility relies on the Sørensen–Dice coefficient applied to character bigrams. For two strings X and Y, the coefficient \( S \) is relative to the intersection of their bigram sets:

\( S = \frac{2 |X \cap Y|}{|X| + |Y|} \)
Formula 2.4: Sørensen–Dice Coefficient

The coefficient is a value between \(0\) and \(1\). In the examResponseRoutes.js, an NLP similarity threshold of 0.75 (75%) is considered sufficient to award full correctness marks for a descriptive question automatically.

2.4 Integrity Scoring Algorithm Validation

The integrity score represents the probabilistic confidence that an exam was taken honestly. It starts at a basal rate of 100% and degrades based on penalization vectors. The weight (\( W \)) of an event (\( E \)) is proportional to its risk to exam integrity.

\( Score_{integrity} = \max\left(0, \quad 100 - \sum_{i=1}^{k} W_i \cdot C(E_i)\right) \)
Formula 2.5: Weighted Integrity Penalty Model

According to IntegrityScoreEngine.js, weights are distributed as follows:

  • Tab Switch: \( W = 8 \)
  • Sustained Stress Spike (>60sec): \( W = 10 \)
  • Copy & Paste Attempt: \( W = 15 \)
  • Face Missing constraint error: \( W = 20 \)
  • Multiple Faces located constraint error: \( W = 25 \)

2.5 Item Response Theory (IRT) in Adaptive Training

The Cognitive Training module differentiates this system from a simple evaluation tool. Using concepts derived from Item Response Theory (specifically Computerized Adaptive Testing - CAT), the sequence of questions is not fixed.

If a user answers correctly, the target difficulty \(\theta\) increments to challenge their capability threshold. If they answer incorrectly, \(\theta\) decreases. Uniquely, this project injects Bio-feedback into the IRT calculation. If the user's localized stress level exceeds \( 60\% \), the system forces \(\theta\) down linearly, preventing cognitive burnout and allowing psychological recovery during training.

3. System Architecture & Design

3.1 Top-Level Architecture Structure

The solution embraces a stateless, decoupled Client-Server architecture utilizing the REST protocol. Data is transmitted purely in formatted JSON payloads. The Edge AI concept is heavily utilized: all facial detection model geometries and inference calculations occur on the Client's hardware (Browser) to ensure data privacy (no webcam feeds exist on servers) and to drastically reduce server overhead.

3.2 Technology Stack Rationale (MERN)

  • MongoDB & Mongoose: Unlike relational SQL data, academic questions, responses, and timeline telemetries vary wildly in structure. MongoDB's BSON structure natively nests arrays of exam questions flawlessly.
  • Express & Node.js: Because the system requires ingesting high-frequency streams of performance snapshots and integrity events from multiple simultaneous students, Node's event-driven, non-blocking I/O model prevents process locking.
  • React.js: The UI requires instant feedback (stress bars, countdowns, warning modals). React’s Virtual DOM isolates these hyper-active rendering cycles, ensuring the browser does not freeze under the simultaneous execution of webcam painting and event handling.

4. Application Workflows & System Processes

4.1 Authentication Pipeline Workflow

The authentication pipeline operates on a Two-Factor logic (Possession of Credentials + Biometric Inheritance):

  1. The User initiates the login handler passing username and password.
  2. The FaceCapture Ref invokes the webcam. The underlying tinyFaceDetector locates the spatial bounding box.
  3. The image chunk is normalized and passed to faceRecognitionNet, producing the 128D descriptor array.
  4. The Client POSTs the payload to the auth route.
  5. The server retrieves the stored descriptor for the username from MongoDB, applies Euclidean Math (Formula 2.1), and conditionally assigns a JSON session token response.

4.2 Automated AI Proctoring Workflow

During an active StartExam.js session, multiple parallel processing threads execute:

  • Thread A (Security Constraints): Event listeners are bound to visibilitychange, fullscreenchange, and keydown. If breached, the count increments. At \(N=3\), the system forcefully executes the submit function irrespective of the timer.
  • Thread B (Bio-Feedback Aggregation): Every 2000ms, the BehaviorMonitor resolves face visibility, EAR calculations, and facial geometry. Every 1000ms, the AnalyserNode calculates FFT arrays for audio.
  • Thread C (Persistence): Every 4000ms, Thread B's smoothed values are grouped into a "Status Snapshot," injected into a local buffer, and backed up to localStorage (Fail-safe against accidental browser crashes).

4.3 Analytics & Intelligence Generation

Post-submission, the PerformanceAnalyticsEngine digests the arrays of telemetry. It segregates data by the exam's topicTag. The system highlights Cognitive Hotspots—topics or specific questions where the user's focus broke or stress averaged \(>70\%\). It also outputs actionable text heuristics (e.g., "Reduce hesitation by practicing time-boxed drills") based on threshold triggers.

6.1 Core Evaluation Logic in Controllers

The most critical code path resides in the /submit route for exams. It loops over student submissions, detects the type, and executes branching logic.

backend/routes/examResponseRoutes.js (Excerpt)
for (const response of responses) {
  const question = exam.questions.find(q => q._id.toString() === response.questionId);
  if (!question) continue;
  
  totalScore += 1;
  let processedResponse = { questionId: response.questionId };

  // Boolean logic for MCQ
  if (question.type === 'multiple_choice') {
    processedResponse.answerIndex = response.answer;
    if (Number(response.answer) === Number(question.correctAnswer)) obtainedScore += 1;
  } 
  // NLP Logic Pipeline for Descriptive texts
  else if (question.type === 'descriptive') {
    processedResponse.descriptiveAnswer = response.answer;
    if (question.referenceAnswer && response.answer) {
      
      // Calculate Lexical Similarity
      const similarity = await computeSimilarity(question.referenceAnswer, response.answer);
      if (similarity >= 0.75) obtainedScore += 1;
      processedResponse.similarity = similarity;
      
      // Cross-reference with Plagiarism external cluster
      const plagiarism = await computePlagiarismScore(response.answer, question.referenceAnswer);
      processedResponse.plagiarism = plagiarism;
      descriptivePlagiarismTotal += plagiarism;
      descriptiveCount++;
    }
  }
  processedResponses.push(processedResponse);
}

6.2 NLP and Similarity Utilities

backend/utils/similarity.js
const stringSimilarity = require('string-similarity');

/**
 * Returns a score [0..1] mapping lexical closeness via Dice's Coefficient.
 */
function computeSimilarity(reference, studentAnswer) {
  const a = String(reference || '');
  const b = String(studentAnswer || '');
  return stringSimilarity.compareTwoStrings(a, b);
}
module.exports = { computeSimilarity };

// Mapping 68 nodes onto coordinate plane
const landmarks = detection.landmarks;
const leftEye = landmarks.getLeftEye();

// Implementation of Formula 2.2 (Eye Aspect Ratio)
const calculateEAR = (eye) => {
  const v1 = Math.sqrt(Math.pow(eye[1].x - eye[5].x, 2) + Math.pow(eye[1].y - eye[5].y, 2));
  const v2 = Math.sqrt(Math.pow(eye[2].x - eye[4].x, 2) + Math.pow(eye[2].y - eye[4].y, 2));
  const h = Math.sqrt(Math.pow(eye[0].x - eye[3].x, 2) + Math.pow(eye[0].y - eye[3].y, 2));
  return (v1 + v2) / (2.0 * h);
};

const avgEAR = (calculateEAR(leftEye) + calculateEAR(landmarks.getRightEye())) / 2;

// Threshold rules for physiological response
if (avgEAR < 0.2) visualStress += 40;     // Eyes significantly closed / sleepy / panic reading
else if (avgEAR < 0.25) visualStress += 15;
